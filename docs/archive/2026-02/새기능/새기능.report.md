## 기술적 의사결정

### framer-motion 선택 이유
- React 선언적 API (`motion.div`, `whileInView`)로 코드 간결성 확보
- SSR 지원 (Next.js App Router 호환)
- `useReducedMotion` 내장으로 접근성 자동 처리

### 클라이언트/서버 컴포넌트 분리 전략
- `app/page.tsx`는 Server Component 유지
- 애니메이션이 필요한 컴포넌트만 `"use client"` 선언
- `AnimatedSection`을 별도 wrapper로 분리하여 Server Component 오염 방지

---

## 학습 포인트

1. **`whileInView` vs Intersection Observer**: framer-motion의 `whileInView`가 Intersection Observer를 추상화하여 코드량 대폭 감소
2. **`viewport={{ once: true }}`**: 섹션이 한 번만 애니메이션 실행되어 스크롤 업 시 재실행 없음 (ISTJ 미니멀 정체성 유지)
3. **접근성 우선**: `useReducedMotion`을 모든 애니메이션 컴포넌트에 일관 적용
4. **설계 명세 품질**: `HeroContent` 사례처럼 설계 문서의 import 예시와 본문 정의가 일치해야 함
